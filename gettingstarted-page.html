Step 0

First things first, it's always necessary to create .gitignore at the root of the project to prevent accidental uploads.

Go to gitignore.io and type in all the necessary items.

For example, these are my go-tos:

  • Git
  • Windows
  • macOS
  • VisualStudioCode
  • SublimeText
  • Node

Copy and paste the result into your own .gitignore file.

.gitignore

This file contains general rules about the files and folders for git to ignore before adding to stage, when preparing to push to remote repository.

For more information, refer to the git documentation.

File Structure

This is how my files are structured:

          
  thisproject/
  ├── .gitignore
  ├── node_modules
  ├── dist
  │   ├── styles.css
  │   └── styles.css.map
  ├── images
  │   └── home.svg
  ├── scss
  │   └── custom.scss
  ├── index.html
  ├── page1.html
  ├── page2.html
  └── page3.html

One thing to note about this is that the node_modules folder will generally be ignored and dist folder will contain the compiled css stylesheets which are going to be referenced in the html.

Also, I've imported the Bootstrap styles in node_modules in my custom.scss, which means I can develop styles in the custom.scss and locally compile with Bootstrap core to styles.css in the dist folder.

Bootstrap Setup

There are two ways to set up Bootstrap:

  1. CDN (Documentation)
  2. NPM Package (View source)

Spoiler Alert

I've eventually chosen the NPM package method as it is essentially the same as CDN while providing more flexibility.

CDN

Copy and paste this stylesheet <link> before all the other stylesheets in the <head>.

          
  <link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/bootstrap/4.5.0/css/bootstrap.min.css" integrity="sha384-9aIt2nRpC12Uk9gS9baDl411NQApFmC26EwAOH8WgZl5MYYxFfc+NcPb1dKGj7Sk" crossorigin="anonymous">

Make sure to include these scripts right before the closing </body> tag.

          
  <script src="https://code.jquery.com/jquery-3.5.1.slim.min.js" integrity="sha384-DfXdz2htPH0lsSSs5nCTpuj/zy4C+OGpamoFVy38MVBnE+IbbVYUew+OrCXaRkfj" crossorigin="anonymous"></script>
  <script src="https://cdn.jsdelivr.net/npm/popper.js@1.16.0/dist/umd/popper.min.js" integrity="sha384-Q6E9RHvbIyZFJoft+2mJbHaEWldlvI9IOYy5n3zV9zzTtmI3UksdQRVvoxMfooAo" crossorigin="anonymous"></script>
  <script src="https://stackpath.bootstrapcdn.com/bootstrap/4.5.0/js/bootstrap.min.js" integrity="sha384-OgVRvuATP1z7JjHLkuOU7Xw704+h835Lr+6QL9UvYjZE3Ipu6Tp75j7Bh/kR0JKI" crossorigin="anonymous"></script>

Important

These scripts will add jQuery, Popper.js, and Bootstrap's own JavaScript to the website, which are required for some features of Bootstrap, e.g. components.

NPM Package

First, I navigated to the root level of the project directory and npm init to get npm set up.

          
  cd ~/go/to/the/project
  npm init -y
-y flag will pass yes to all the initialization questions and lets you complete quickly.

Install bootstrap

          
  npm install bootstrap

Import the source Sass files into custom.scss, I've chosen option A here.

          
  // Option A: Include all of Bootstrap
  @import "../node_modules/bootstrap/scss/bootstrap";

  
  // Option B: Include parts of Bootstrap
  // Required
  @import "../node_modules/bootstrap/scss/functions";
  @import "../node_modules/bootstrap/scss/variables";
  @import "../node_modules/bootstrap/scss/mixins";
  // Optional
  @import "../node_modules/bootstrap/scss/reboot";
  @import "../node_modules/bootstrap/scss/type";
  @import "../node_modules/bootstrap/scss/images";
  @import "../node_modules/bootstrap/scss/code";
  @import "../node_modules/bootstrap/scss/grid";

Note

The advantage of this method is the ability to access the core stylesheets when applying customizations, as opposed to overriding compiled styles in CDN.

This is the recommended way by Bootstrap, but considering that Webpack and the build process is out of the scope for this project, I've chosen to build locally.

Check out this question on stackoverflow for some discussion regarding this.

Now run this command to build the stylesheets locally with --watch to listen for changes, note that save triggers a rebuild.

          
  sass --watch scss/custom.scss:dist/styles.css

Important!

Since we are referencing the css file in the distfolder, we want to push the dist folder to remote by removing dist from .gitignore if it exists.
